# 传统古建油饰地仗灰层配比与披灰工序生产力系统

## 项目简介

本系统是面向古建木构地仗施工的专业桌面应用，旨在实现传统古建筑油饰地仗工艺的数字化管理，涵盖灰层配比计算、施工工序管控、质量风险预警、工艺档案追溯等核心功能。

## 功能模块

### 1. 构件录入页
- 录入木构件基本信息（名称、编号、材质、位置、尺寸）
- 记录表面状况检测（裂缝、腐朽、松动、平整度、含水率）
- 选择目标地仗等级（一麻五灰、二麻五灰等7种传统做法）
- 自动计算构件表面积
- 构件状态管理（待施工、施工中、已完成）

### 2. 灰层配比页
- 根据地仗等级自动生成各灰层（捉缝灰、通灰、中灰、细灰、灰）的设计配比
- 录入实际配比，自动计算与设计值的偏差
- 偏差超过±5%自动标红警示，超过±10%触发高危预警
- 麻布/麻丝用量计算与搭接参数配置
- 按构件面积反推各灰层用料，生成备料清单
- 配比分布图表可视化展示

### 3. 披灰工序页
- 施工环境温湿度模拟，实时计算对油满灰料固化的影响
- 显示固化质量指数、固化速率系数、环境风险等级
- 各灰层干燥时间自动计算与调整（基于温湿度）
- 工序时间线管理，灰层干燥状态实时追踪
- 灰层未干即上层的风险检测与预警
- 干燥时间衔接校验，避免起壳脱落
- 施工记录录入（施工人员、实际厚度、备注）

### 4. 工艺档案页
- 已完成工序的工艺档案管理
- 施工记录追溯（各层施工时间、人员、配比、环境条件）
- 质量评级展示
- 材料用量统计
- 档案导出功能

### 5. 范式库页
- 内置7种传统地仗做法标准范式
- 支持自定义创建新的工艺范式
- 范式复制、编辑、删除管理
- 各灰层配比、厚度、干燥时间的详细配置
- 适用材质与施工条件标注

## 核心算法

### 灰层配比算法
- 基于传统工艺的灰:灰:桐油:水比例体系
- 根据表面状况自动调整灰层厚度系数
- 裂缝宽度>5mm增加15%用量，腐朽增加10%，平整度差增加10%

### 干燥时间模型
```
温度系数：<10°C=2.5, 10-15°C=1.8, 15-20°C=1.3, 20-25°C=1.0, 25-30°C=0.8, 30-35°C=0.6, >35°C=0.5
湿度系数：<40%=0.8, 40-60%=1.0, 60-75%=1.3, 75-85%=1.8, >85%=2.5
调整后干燥时间 = 标准时间 × 温度系数 × 湿度系数
```

### 风险预警机制
- 配比偏差：单组分偏差>5%标黄，>10%标红
- 干燥时间：前层干燥进度<80%即施工触发预警
- 环境条件：温湿度超出最佳范围给出施工建议

### 材料用量计算
```
体积 = 面积 × 厚度
灰用量 = 体积 × 1.8 × (灰比例/总比例)
灰用量 = 体积 × 2.2 × (灰比例/总比例)
桐油用量 = 体积 × 0.15 × (油比例/总比例)
麻丝用量 = 面积 × 0.08 × (1 + 搭接量/幅宽)
麻布用量 = 面积 × 1.1 × (1 + 搭接量/幅宽)
```

## 传统工艺参考

本系统基于以下传统古建油饰地仗做法：

| 做法名称 | 灰层数 | 麻/布层数 | 适用场景 |
|---------|--------|----------|---------|
| 一麻五灰 | 5层灰 | 1层麻 | 重要构件、官式建筑 |
| 二麻五灰 | 5层灰 | 2层麻 | 次重要构件 |
| 三麻三布灰 | 5层灰 | 3层麻+3层布 | 大型构件 |
| 一麻布灰 | 5层灰 | 1层麻+1层布 | 中等要求 |
| 一五灰 | 5层灰 | 无 | 一般要求 |
| 单三布灰 | 2层灰 | 3层布 | 简易做法 |
| 三灰 | 3层灰 | 无 | 最简易做法 |

## 特色功能

1. **偏差识别**：自动比对实际配比与设计配比，超出阈值标红警示
2. **环境模拟**：实时模拟温湿度对灰料固化的影响，给出施工建议
3. **风险预警**：对灰层未干即上层等违规操作实时预警
4. **用量反推**：根据构件面积自动计算各材料用量，生成备料清单
5. **工艺追溯**：完整记录每道工序的施工信息，建立工艺档案
6. **范式管理**：内置标准工艺，支持自定义扩展新做法

## 注意事项

- 最佳施工温度：15°C - 25°C
- 最佳施工湿度：45% - 65%
- 各灰层必须完全干燥后方可进行下一道工序
- 建议材料备料增加10%损耗量
- 施工前请确认构件表面含水率<15%
